## 5. 代码规范

### **通用规范**

- **文件命名**：
  - 根据项目语言和框架约定选择命名风格
  - 保持项目内命名风格一致

- **目录结构**：
  - 按功能模块划分目录
  - 相关文件放在同一目录下
  - 公共代码放在共享目录

- **代码风格**：
  - 使用代码格式化工具统一风格
  - 使用代码检查工具保证质量
  - 编写清晰的注释和文档

### **前端代码规范**

- **命名约定**：
  - 组件：根据框架约定（如 PascalCase 或 kebab-case）
  - 变量/函数：camelCase
  - 常量：UPPER_SNAKE_CASE
  - 文件：根据项目约定

- **目录约定**：
  - 组件目录包含主文件和样式文件
  - 页面目录包含主文件、私有组件和样式文件

### **后端代码规范**

- **命名约定**：
  - 类：PascalCase
  - 函数/变量：根据语言约定（如 camelCase 或 snake_case）
  - 常量：UPPER_SNAKE_CASE
  - 文件：根据语言约定

- **目录约定**：
  - 按功能模块组织代码
  - 每个模块包含必要的初始化文件
  - 相关功能放在同一目录下
