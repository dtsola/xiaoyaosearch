# 架构决策 - 前端首页接口对接与功能简化

**决策日期**：2024年01月20日

## 背景
为完成小遥搜索前端首页与后端API的接口对接，简化用户界面复杂度，提升用户体验，需要明确功能边界、统一技术规范，并制定详细的实施方案。

## 解决方案
1. **保留核心搜索功能**：文本搜索、多模态搜索、搜索建议、搜索历史等核心功能完全对接真实API
2. **简化文件操作**：移除文件夹选择、文件收藏、文件删除功能，文件预览/打开统一为操作系统默认应用打开
3. **统一技术规范**：前后端枚举值统一使用后端规范，确保类型安全和数据一致性
4. **新增状态聚合接口**：为底部状态栏提供统一的状态查询接口，聚合索引、数据、搜索统计等信息

## 影响
1. **前端开发**：减少30%的UI复杂度，聚焦核心搜索体验，降低开发和维护成本
2. **后端开发**：新增1个状态聚合接口，简化API设计，提高系统稳定性
3. **用户体验**：简化操作流程，提高搜索效率，符合桌面应用使用习惯
4. **技术债务**：统一枚举规范，避免类型不匹配问题，提升代码质量
